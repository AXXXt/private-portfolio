# Personal Portfolio Architecture

> Stack: Next.js App Router, React 19, Tailwind CSS, shadcn/ui, GSAP.
> Goal: keep the portfolio fast, accessible, animation-rich, and predictable for human and AI contributors.

## 1. Architecture Principles

- Prefer Server Components by default. Add `"use client"` only for interactivity, browser APIs, animation, or stateful UI.
- Keep route files small. `app/**/page.tsx` composes sections; section internals live beside the route or in shared components.
- Treat shadcn/ui as editable source code, not a black-box dependency.
- All GSAP animation code must be scoped, cleanup-safe, and responsive-motion aware.
- Optimize for controlled creative expression: strong visuals, but no animation that damages readability, accessibility, or Core Web Vitals.
- Every AI-generated change must preserve directory boundaries and pass lint/type/build checks before merge.

### 2.1 App Router Rules

- `app/layout.tsx`: root HTML, metadata defaults, font setup, global providers.
- `app/page.tsx`: home route composition only. Avoid embedding large business sections directly.
- `app/(site)/**`: route groups for organization without changing URLs.
- `app/**/_components`: route-private components that must not be imported outside that route segment.
- `app/**/loading.tsx`, `error.tsx`, `not-found.tsx`: route-local UX states.
- `app/**/route.ts`: API endpoints only. Do not colocate UI helpers here.

### 2.2 Component Boundaries

| Type | Path | Purpose | Import Rules |
| --- | --- | --- | --- |
| shadcn/ui atoms | `components/ui/*` | Editable primitive components generated by shadcn CLI | Import anywhere; no business logic |
| Global shared components | `components/global/*` | Header, footer, logo, theme controls, reusable navigation | Import across routes |
| Reusable page sections | `components/sections/*` | Portfolio sections reused across pages | May use UI atoms and motion wrappers |
| Motion wrappers | `components/motion/*` | Generic animated wrappers and interaction components | Client components only |
| Route-private components | `app/**/_components/*` | Components coupled to one route's content or layout | Import only inside that route segment |
| Animation modules | `animations/*` and `hooks/animations/*` | GSAP registration, shared timelines, hooks | No JSX in `animations/*` |

## 3. React And Next.js Standards

### 3.1 Server And Client Components

- Default to Server Components for static content, data loading, metadata, and project pages.
- Use Client Components only when using:
  - GSAP or browser APIs.
  - shadcn interactive components requiring client behavior.
  - Local UI state, gestures, hover intent, media queries, or event handlers.
- Pass serializable data from Server Components into Client Components.
- Keep `"use client"` files as leaf components where possible.

### 3.2 Metadata And Assets

- Use `generateMetadata` for project detail pages.
- Use `next/image` for raster images and remote images when optimization is required.
- Store portfolio media in `public/images`, `public/video`, and `public/models`.
- Use descriptive asset names:
  - `project-name-cover.avif`
  - `project-name-motion-demo.mp4`
  - `portrait-hero.webp`

## 4. GSAP Animation Standards

### 4.1 Non-Negotiable Rules

- All React animation must use `@gsap/react` and `useGSAP()`.
- Every file using GSAP must be a Client Component.
- Register GSAP plugins in one shared module or at the top of the client module.
- Do not run GSAP in Server Components, `layout.tsx` server files, or plain module scope except plugin registration.
- Do not use broad `document.querySelector` or `document.querySelectorAll` inside components.
- Use `useRef` plus `useGSAP({ scope })` so selector text is limited to the current component subtree.
- Prefer class/data selectors inside the scoped root:
  - Good: `.js-card`, `[data-animate='title']`
  - Bad: `section > div:nth-child(3)`, global `.button`, global `main .card`
- Every ScrollTrigger must be created inside `useGSAP()` so it is reverted on cleanup.

### 4.2 Shared GSAP Setup

```ts
// animations/gsap.ts
"use client";

import { gsap } from "gsap";
import { useGSAP } from "@gsap/react";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.registerPlugin(useGSAP, ScrollTrigger);

gsap.defaults({
  ease: "power3.out",
  duration: 0.7,
});

export { gsap, ScrollTrigger };
```

### 4.3 Standard Component Template

```tsx
// components/sections/hero-section.tsx
"use client";

import { useRef } from "react";
import { gsap, ScrollTrigger } from "@/animations/gsap";
import { useGSAP } from "@gsap/react";

export function HeroSection() {
  const rootRef = useRef<HTMLElement | null>(null);

  useGSAP(
    () => {
      const mm = gsap.matchMedia();

      mm.add("(prefers-reduced-motion: reduce)", () => {
        gsap.set("[data-animate]", { clearProps: "all", autoAlpha: 1 });
      });

      mm.add("(max-width: 767px)", () => {
        gsap.from("[data-animate='headline']", {
          y: 20,
          autoAlpha: 0,
          duration: 0.45,
        });
      });

      mm.add("(min-width: 768px)", () => {
        const tl = gsap.timeline({
          defaults: { ease: "power3.out" },
          scrollTrigger: {
            trigger: rootRef.current,
            start: "top 80%",
            end: "bottom top",
            scrub: false,
          },
        });

        tl.from("[data-animate='headline']", {
          y: 48,
          autoAlpha: 0,
          duration: 0.8,
        }).from(
          "[data-animate='card']",
          {
            y: 32,
            autoAlpha: 0,
            stagger: 0.08,
            duration: 0.65,
          },
          "-=0.35",
        );
      });

      return () => mm.revert();
    },
    { scope: rootRef },
  );

  return (
    <section ref={rootRef} className="relative min-h-svh overflow-hidden">
      <h1 data-animate="headline">Creative Frontend Engineer</h1>
      <div className="grid gap-4 md:grid-cols-3">
        <article data-animate="card" />
        <article data-animate="card" />
        <article data-animate="card" />
      </div>
    </section>
  );
}
```

### 4.4 ScrollTrigger Template

```tsx
useGSAP(
  () => {
    const timeline = gsap.timeline({
      scrollTrigger: {
        trigger: "[data-scroll-section]",
        start: "top 70%",
        end: "bottom 20%",
        scrub: 0.6,
        invalidateOnRefresh: true,
      },
    });

    timeline
      .to("[data-layer='foreground']", { yPercent: -12, ease: "none" })
      .to("[data-layer='background']", { yPercent: 8, ease: "none" }, 0);
  },
  { scope: rootRef },
);
```


### 4.6 Performance Rules

Animate these properties by default:

- `x`, `y`, `z`, `xPercent`, `yPercent`
- `scale`, `scaleX`, `scaleY`
- `rotation`, `rotate`
- `opacity` or GSAP `autoAlpha`
- CSS variables only when they do not force layout on every frame

Avoid or require explicit approval for:

- `width`, `height`, `top`, `right`, `bottom`, `left`
- `margin`, `padding`, `border-width`
- `font-size`, `line-height`, `letter-spacing`
- heavy `filter`, `backdrop-filter`, large blur, or repeated box-shadow changes

Rules:

- Use transform-based illusions instead of layout animation. Example: animate `scaleX` instead of `width`.
- Do not animate layout properties during scroll unless profiling proves it is acceptable.
- Use `will-change` only shortly before animation and remove it after completion.
- Do not leave `will-change` permanently on large sections, full-screen media, or many repeated cards.
- For large image/video sections, prefer fixed dimensions, `aspect-ratio`, and transform-only parallax.
- Use `ScrollTrigger.refresh()` after async media/layout changes that affect trigger positions.

```tsx
useGSAP(
  () => {
    gsap.set("[data-card]", { willChange: "transform, opacity" });

    gsap.from("[data-card]", {
      y: 32,
      autoAlpha: 0,
      stagger: 0.08,
      onComplete: () => {
        gsap.set("[data-card]", { willChange: "auto" });
      },
    });
  },
  { scope: rootRef },
);
```

### 4.7 Responsive Motion

- Tailwind controls layout; `gsap.matchMedia()` controls motion behavior.
- Mobile motion must be shorter, less layered, and less scroll-scrubbed.
- Desktop may use pinned sections, parallax, and multi-step timelines when content remains readable.
- Always include a reduced-motion branch.

Breakpoint mapping:

| Context | Tailwind | GSAP media query |
| --- | --- | --- |
| Mobile default | unprefixed | `(max-width: 767px)` |
| Tablet and up | `md:` | `(min-width: 768px)` |
| Desktop and up | `lg:` | `(min-width: 1024px)` |
| Wide desktop | `xl:` | `(min-width: 1280px)` |
| Reduced motion | `motion-reduce:` | `(prefers-reduced-motion: reduce)` |

## 5. shadcn/ui And Tailwind CSS Standards

### 5.1 shadcn/ui Ownership

- CLI-generated shadcn components live only in `components/ui`.
- Treat `components/ui` as editable project source.
- Keep shadcn atoms business-agnostic:
  - No project copy.
  - No route imports.
  - No analytics calls.
  - No hard-coded portfolio content.
- Business variants belong in `components/global`, `components/sections`, or `app/**/_components`.

### 5.2 AI Styling Rules

- If a base UI primitive needs a style change, edit the source in `components/ui`.
- Do not override shadcn primitives from business components using brittle wrapper selectors.
- Prefer `class-variance-authority` variants for systematic style options.
- Use `cn()` for conditional Tailwind classes.
- Do not add global CSS unless the style is truly global: theme tokens, typography defaults, selection color, scroll behavior, or custom utilities.

Good:

```tsx
// components/ui/button.tsx
const buttonVariants = cva(
  "inline-flex items-center justify-center rounded-md text-sm font-medium transition-colors",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        portfolio: "bg-foreground text-background hover:bg-foreground/90",
      },
    },
  },
);
```

Avoid:

```tsx
// Do not force primitive internals from business code.
<div className="[&_.shadcn-button]:!rounded-none [&_.shadcn-button]:!bg-black">
  <Button />
</div>
```

### 5.3 Ref Exposure For GSAP

Use `forwardRef` for shadcn/Radix-compatible primitives and motion wrappers so GSAP can animate the actual DOM node.

```tsx
// components/motion/animated-button.tsx
"use client";

import * as React from "react";
import { Button, type ButtonProps } from "@/components/ui/button";

export const AnimatedButton = React.forwardRef<HTMLButtonElement, ButtonProps>(
  function AnimatedButton({ className, ...props }, ref) {
    return <Button ref={ref} className={className} {...props} />;
  },
);
```

For component-internal micro-interactions, integrate `useGSAP()` inside the client wrapper:

```tsx
"use client";

import { useRef } from "react";
import { useGSAP } from "@gsap/react";
import { gsap } from "@/animations/gsap";
import { Button } from "@/components/ui/button";

export function MagneticButton(props: React.ComponentProps<typeof Button>) {
  const rootRef = useRef<HTMLButtonElement | null>(null);

  useGSAP(
    () => {
      const el = rootRef.current;
      if (!el) return;

      const enter = () => gsap.to(el, { scale: 1.04, duration: 0.18 });
      const leave = () => gsap.to(el, { scale: 1, duration: 0.22 });

      el.addEventListener("pointerenter", enter);
      el.addEventListener("pointerleave", leave);

      return () => {
        el.removeEventListener("pointerenter", enter);
        el.removeEventListener("pointerleave", leave);
      };
    },
    { scope: rootRef },
  );

  return <Button ref={rootRef} {...props} />;
}
```

### 5.4 Tailwind Rules

- Use mobile-first classes: unprefixed for mobile, then `sm:`, `md:`, `lg:`, `xl:`, `2xl:`.
- Prefer container constraints: `mx-auto max-w-* px-*`.
- Use `@container` for component-level responsive behavior when parent size matters more than viewport size.
- Keep class order readable:
  - layout
  - spacing
  - sizing
  - typography
  - color
  - effects
  - state
  - responsive variants
- Use design tokens via CSS variables for colors and radii. Avoid hard-coded one-off hex values in components.
- Use `motion-safe:` and `motion-reduce:` for CSS transitions.

## 6. Accessibility And UX

- Interactive shadcn/Radix components must preserve keyboard behavior and ARIA attributes.
- Do not remove focus rings. Restyle them with tokens if needed.
- All animated content must remain readable while moving.
- Motion must not be the only way to understand hierarchy or state.
- Provide `alt` text for meaningful images. Decorative images use empty alt text.
- Ensure color contrast holds in dark and light themes.
- Avoid scroll hijacking. Pinned sections must not trap users or break normal page progress.

## 7. Code Quality

### 7.1 TypeScript

- Use TypeScript for all app code.
- Export explicit prop types for reusable components.
- Avoid `any`; use `unknown` plus narrowing when necessary.
- Use `satisfies` for config and structured content.

```ts
export const projects = [
  {
    slug: "motion-system",
    title: "Motion System",
    year: "2026",
  },
] satisfies Array<{
  slug: string;
  title: string;
  year: string;
}>;
```

### 7.2 Imports

- Use `@/*` alias for app imports.
- No deep imports from another route's `_components`.
- Animation utilities import from `@/animations/*`.
- shadcn atoms import from `@/components/ui/*`.

### 7.3 Checks Before Merge

Run before merging:

```bash
npm run lint
npm run typecheck
npm run build
```

For animation-heavy changes, also verify:

- No hydration errors.
- No console warnings.
- No lingering ScrollTriggers after route transitions.
- No layout shift during entrance animations.
- Reduced-motion mode is acceptable.
- Mobile scroll remains smooth.

### 8.3 Git Commit Rules

- Commit before any broad AI-assisted refactor.
- Commit before adding complex ScrollTrigger, pinned sections, or page-wide timelines.
- Commit before changing `components/ui/*` primitives used across the site.
- Prefer small, reversible commits:
  - `feat(hero): add scoped GSAP entrance timeline`
  - `refactor(ui): tune button variants`
  - `perf(work): replace width animation with scale transform`
  - `docs: add animation standards`
- Do not mix unrelated work:
  - No UI primitive refactor plus content rewrite in one commit.
  - No animation system change plus routing change in one commit.
  - No dependency upgrade inside feature commits.

### 8.4 Branching

- Use short feature branches:
  - `feat/hero-motion`
  - `feat/project-gallery`
  - `refactor/shadcn-theme`
- For experimental animation, create a dedicated branch and keep commits especially small.
- If an AI-generated change feels risky, stop after file creation and review before asking for iteration.

## 9. Dependency Policy

- Core animation library is GSAP. Do not add Framer Motion, Motion One, or anime.js without an explicit architecture decision.
- shadcn/ui is the component source pattern. Do not add a second full component library.
- Prefer native CSS and Tailwind utilities before adding micro-libraries.
- Any dependency must justify:
  - bundle cost
  - maintenance state
  - accessibility impact
  - whether it works with React Server Components
